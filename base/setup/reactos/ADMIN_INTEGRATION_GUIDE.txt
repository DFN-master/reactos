/*
 * ADMINISTRATOR ACCOUNT PAGE FEATURES
 * ===================================
 * 
 * Input Fields:
 *   - Username: Default "Administrator", alphanumeric + underscore
 *   - Password: Hidden with bullets (●), minimum 4 recommended 8 chars
 *   - Confirm Password: Must match first password
 *   - Show Password: Checkbox to reveal password during typing
 * 
 * Validation Rules:
 *   - Username: Required, no special characters
 *   - Password: Optional (empty allowed for test), 4-255 chars
 *   - Passwords must match
 *   - Feedback messages for invalid input
 * 
 * Security Features:
 *   - Password strength indicator (progress bar)
 *   - Hidden password display by default
 *   - Password length requirements
 *   - No password hints displayed
 */

/*
 * TESTING GUIDE
 * =============
 * 
 * Unit Tests for modern_admin_page.c:
 * 
 * 1. ValidateUsername()
 *    Input: "Administrator" → Expected: TRUE
 *    Input: "user123" → Expected: TRUE
 *    Input: "user@domain" → Expected: FALSE (@ not allowed)
 *    Input: "" → Expected: FALSE (empty)
 *    Input: "admin123!@#" → Expected: FALSE (special chars)
 * 
 * 2. ValidatePassword()
 *    Input: "MyPassword123" → Expected: TRUE
 *    Input: "short" → Expected: TRUE (>=4 allowed)
 *    Input: "" → Expected: TRUE (empty allowed)
 *    Input: Very long password... → Expected: FALSE if >255 chars
 * 
 * 3. CreateUserAccount()
 *    Create account with username="test", password="test123"
 *    Verify in registry: HKLM\SAM\SAM\Domains\Builtin\Users\Names\test
 *    Verify RID created: HKLM\SAM\SAM\Domains\Builtin\Users\1000 (or next)
 * 
 * 4. AdminAccountPageProc() Dialog
 *    - Username field accepts input correctly
 *    - Password field shows bullets (●) by default
 *    - Show Password checkbox reveals/hides text
 *    - Next button disabled until passwords match
 *    - Error messages display for invalid input
 *    - Dialog remembers values when navigating back
 */
